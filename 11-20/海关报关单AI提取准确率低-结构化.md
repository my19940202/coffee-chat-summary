# 海关报关单AI提取准确率低 - 行业痛点分析

## 📋 背景介绍
一位在进出口海关贸易软件行业深耕10+年的从业者，分享了一个真实业务挑战案例。

## 🎯 业务场景

### 核心业务
- **客户来源**：各行各业的不同客户
- **处理对象**：客户发来的各种货物报关单（PDF格式）
- **业务目标**：将非结构化的PDF报关单，整理成结构化数据，最终生成符合海关规范的报关单

### 报关单包含的信息
- 货物名称
- 发货时间
- 公司名称
- 货物规格（重量、体积等）
- 货物名称对应的海关ID序号
- 其他大量相关信息

## 💡 技术方案

### 当前实现方式
1. **技术栈**：调用大模型API
2. **处理流程**：直接将PDF发给大模型
3. **提取方式**：通过编写Prompt提示词进行信息抽取
4. **特点**：需要提取的信息过多，Prompt写得很长

### 历史尝试
- 对于固定公司、规则固定的情况：可以用规则引擎结构化提取
- 但面对多样化客户时：规则方案失效

## ⚠️ 遇到的问题

### 准确率不稳定
- **表现好的情况**：准确率可达90%+
- **表现差的情况**：准确率仅50%-60%
- **整体表现**：忽高忽低，不够稳定

### 已尝试的优化方案
- ✅ 尝试过多个大模型
- ✅ 尝试过OCR API
- ❌ **结果**：整体表现仍不够理想

## 🔥 核心痛点

1. **准确率不够高**：无法达到可售卖的行业解决方案标准
2. **客户多样性**：不同行业、不同格式的报关单，难以统一处理
3. **信息提取复杂**：需要提取的信息字段过多，Prompt过长
4. **稳定性不足**：准确率波动大，无法保证一致性

## 💭 思考与启发

### 可能的解决方向
- 是否需要结合OCR + 大模型的混合方案？
- 是否需要针对不同行业/客户类型做分类处理？
- 是否需要引入人工审核环节？
- 是否需要建立行业模板库？
